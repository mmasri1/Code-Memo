## Well-Architected Framework

AWS Well-Architected is a checklist mindset: tradeoffs made explicit.

## The pillars

| Pillar | What to ask |
| --- | --- |
| Operational Excellence | can we run/operate/change safely? |
| Security | least privilege, encryption, detection |
| Reliability | blast radius, failure recovery |
| Performance Efficiency | right resource + right scaling |
| Cost Optimization | cost visibility + eliminate waste |
| Sustainability | efficient usage, reduce waste |

## Practical quick wins

- Add budgets + tagging
- Reduce NAT usage with VPC endpoints
- Set log retention
- Use autoscaling and rightsizing
- Prefer managed services where they reduce toil
